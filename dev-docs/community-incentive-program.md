# EST Framework 社区贡献者激励计划

**版本**: 1.0  
**日期**: 2026-03-10  
**状态**: 正式版

---

## 1. 概述

### 1.1 计划目标

EST Framework社区贡献者激励计划旨在建立一个健康、活跃、可持续的开源社区生态系统，通过多层次的激励机制，鼓励和认可社区成员参与项目贡献，共同推动EST Framework的发展。

**核心目标**:
- 建立清晰的贡献者等级体系
- 设计公平的贡献积分系统
- 设计荣誉徽章体系
- 建立贡献排行榜
- 组织定期分享会
- 提供有价值的回馈和奖励

---

## 2. 贡献者等级体系

### 2.1 等级定义

| 等级 | 名称 | 图标 | 所需积分 | 权限 |
|------|------|------|---------|------|
| 1 | 新手 (Newbie) | 🌱 | 0-99 | 基础贡献权限 |
| 2 | 参与者 (Contributor) | 🌿 | 100-499 | Issue创建、PR提交 |
| 3 | 活跃者 (Active) | 🌳 | 500-1999 | PR审核参与、文档维护 |
| 4 | 专家 (Expert) | 🌟 | 2000-4999 | 代码审核、模块维护 |
| 5 | 核心 (Core) | ⭐ | 5000-9999 | 架构设计、决策参与 |
| 6 | 领袖 (Leader) | 👑 | 10000+ | 项目管理、方向决策 |

### 2.2 等级晋升机制

**晋升条件:
- 达到对应等级所需积分
- 满足贡献质量要求
- 通过社区审核（核心成员投票（Expert及以上）

**降级机制**:
- 连续6个月无贡献降1级
- 严重违反社区规范可直接降级

## 3. 贡献积分系统

### 3.1 积分获取方式

| 贡献类型 | 积分 | 说明 |
|---------|------|------|
| **代码贡献** | | |
| 提交Bug修复PR | 50-200 | 根据复杂度和影响 |
| 提交新功能PR | 200-1000 | 根据复杂度和影响 |
| 代码审查 | 10-50 | 每次有效审查 |
| **文档贡献** | | |
| 修复文档错误 | 20-100 | 根据影响范围 |
| 编写新文档 | 100-500 | 根据文档复杂度 |
| 翻译文档 | 100-300 | 根据翻译量 |
| **测试贡献** | | |
| 编写单元测试 | 30-100 | 每个测试文件 |
| 编写集成测试 | 50-200 | 每个测试套件 |
| 发现并报告Bug | 20-100 | 根据Bug严重程度 |
| **社区贡献** | | |
| 回答问题 | 10-50 | 每个被采纳的回答 |
| 参与讨论 | 5-20 | 每次有价值的讨论 |
| 组织分享会 | 200-500 | 每次分享会 |
| 撰写技术博客 | 100-300 | 每篇技术文章 |
| **其他贡献** | | |
| 设计改进建议被采纳 | 50-200 | 根据建议价值 |
| 推广项目 | 50-200 | 根据推广效果 |
| 提供反馈 | 10-50 | 每次有价值的反馈 |

### 3.2 积分计算规则

**PR积分乘数**:
- 基础积分 × 复杂度系数
  - 简单: ×1
  - 中等: ×1.5
  - 复杂: ×2
  - 非常复杂: ×3

**质量系数**:
- 优秀: ×1.5
- 良好: ×1.2
- 合格: ×1
- 需要改进: ×0.5

**示例计算:
```
最终积分 = 基础积分 × 复杂度系数 × 质量系数
```

### 3.3 积分记录和查询

**积分记录**:
- 实时更新
- 贡献详情可查
- 月度/年度统计
- 积分历史追溯

**查询方式**:
- Web界面个人中心
- CLI工具查询
- API接口查询
- 定期邮件通知
