## 🚀 入门指南

### Q: EST Framework 是什么？
A: EST Framework 是一个零依赖的现代Java企业级框架，提供完整的企业应用开发解决方案，包括：
- 核心容器与依赖注入
- Web框架与路由
- 数据访问层
- 安全认证与授权
- 可观测性（Metrics、Logs、Traces）
- AI原生集成
- 多语言SDK支持

### Q: EST Framework 的核心优势是什么？
A: 主要优势包括：
- ✅ **零依赖核心**：核心框架无外部依赖
- ✅ **模块化设计**：按需引入，轻量灵活
- ✅ **API/Impl分离**：清晰的架构层次
- ✅ **AI原生**：内置AI助手和代码生成
- ✅ **云原生**：完整的Docker、K8s、Serverless支持
- ✅ **多语言**：Python、Go、TypeScript、Kotlin SDK

### Q: 如何开始使用EST Framework？
A: 快速开始步骤：
1. 克隆项目：`git clone https://github.com/idcu/est.git`
2. 编译项目：`mvn clean install -DskipTests`
3. 运行示例：查看 `est-examples/` 目录
4. 使用脚手架：`est-cli create project my-app`

### Q: EST Framework 支持哪些Java版本？
A: 最低要求Java 21+，推荐使用Java 21或更高版本以充分利用现代Java特性。

## 🏗️ 架构与设计

### Q: EST Framework 的架构层次是怎样的？
A: 采用六层架构设计：
1. **基础层** (est-base)：工具、集合、格式
2. **核心层** (est-core)：容器、配置、生命周期、AOP、事务
3. **模块层** (est-modules)：功能模块（缓存、事件、安全、数据等）
4. **应用层** (est-app)：Web应用、管理后台、控制台
5. **工具层** (est-tools)：CLI、代码生成、迁移、脚手架
6. **示例层** (est-examples)：示例模块

### Q: 什么是API/Impl分离模式？
A: 每个模块都分为API和Impl两部分：
- **API模块**：定义接口、注解、数据模型
- **Impl模块**：提供具体实现
- 优势：清晰的契约、易于测试、可替换实现

### Q: 如何创建自定义模块？
A: 创建自定义模块的步骤：
1. 创建模块目录结构
2. 定义API接口和模型
3. 实现具体功能
4. 编写单元测试
5. 在父POM中注册模块
6. 详细参考：[模块认证标准](module-certification-standards.md)

### Q: EST Framework 使用什么设计模式？
A: 主要设计模式：
- **依赖注入**：核心容器
- **工厂模式**：服务创建
- **观察者模式**：事件系统
- **策略模式**：可插拔实现
- **构建者模式**：配置构建
- **装饰器模式**：功能增强
- 完整的设计模式实现见：`est-patterns` 模块

## 💻 开发与部署

### Q: 如何编译EST Framework？
A: 常用Maven命令：
```bash
# 完整编译（跳过测试）
mvn clean install -DskipTests

# 编译并运行测试
mvn clean install

# 仅编译
mvn compile

# 代码质量检查
mvn checkstyle:check
```

### Q: 如何运行EST应用？
A: 多种运行方式：
1. **Maven**：`mvn exec:java -Dexec.mainClass="..."`
2. **JAR包**：`java -jar app.jar`
3. **Docker**：`docker-compose up`
4. **Kubernetes**：`kubectl apply -f k8s/`

### Q: 有哪些配置选项？
A: 主要配置方式：
- **YAML配置**：`application.yml`
- **程序化配置**：API构建器
- **环境变量**：`EST_*` 前缀
- **配置文件位置**：`src/main/resources/`

### Q: 如何部署到生产环境？

## 📦 模块使用

### Q: 如何使用依赖注入容器？
A: 基本用法：
```java
// 创建容器
Container container = DefaultContainer.create();

// 注册服务
container.register(MyService.class);

// 获取服务
MyService service = container.get(MyService.class);

// 构造函数注入
@Inject
public MyClass(MyDependency dep) {
    this.dep = dep;
}
```

### Q: 数据访问层如何使用？
A: 使用步骤：
1. 定义实体类（带注解）
2. 创建Repository接口
3. 配置数据源
4. 执行CRUD操作
```java
// 定义实体
@Entity(tableName = "users")
public class User {
    @Id(autoGenerated = true)
    private Long id;
    private String name;
}

// 使用Repository
Repository<User, Long> repo = JdbcRepository.create(dataSource);
User user = repo.findById(1L);
```

### Q: 如何使用事件系统？
A: 事件系统使用：
```java
// 定义事件
public class UserCreatedEvent {
    private final User user;
}

// 发布事件
EventBus.publish(new UserCreatedEvent(user));

// 订阅事件
@EventListener
public void onUserCreated(UserCreatedEvent event) {
    // 处理事件
}
```

### Q: 安全认证如何配置？
A: 安全配置：
```java
// 配置安全
SecurityConfig config = SecurityConfig.builder()
    .jwt()
        .secret("my-secret")
        .expiration(3600)
    .and()
    .rbac()
        .enabled(true)
    .build();

// 使用SecurityManager
SecurityManager manager = SecurityManager.create(config);
```

## 🔧 故障排除

### Q: 编译错误如何解决？
A: 常见编译错误：
1. **POM XML解析错误**：检查嵌套注释
2. **依赖缺失**：运行 `mvn clean install`
3. **Java版本不匹配**：确保Java 21+
4. **具体修复**：参考[短期实施总结](short-term-implementation-summary.md)

### Q: 运行时错误如何调试？
A: 调试步骤：
1. 启用调试日志
2. 检查配置文件
3. 验证依赖版本
4. 使用IDE调试器
5. 查看堆栈跟踪
6. 日志位置：`logs/application.log`

### Q: 内存问题如何处理？
A: 内存优化：
1. 调整JVM堆大小：`-Xmx4g -Xms2g`
2. 启用GC日志
3. 分析堆转储：`jmap -dump:live,file=heap.hprof <pid>`
4. 使用内存分析工具：Eclipse MAT、YourKit
5. 检查内存泄漏
