## 什么是 EST Data？

### 用大白话理解

EST Data 就像是一个"超级档案管理员"。想象一下你有很多数据需要存储：用户信息、产品信息、订单记录...

**传统方式**：你需要自己写代码来管理这些数据的增删改查，每种数据类型都要写一遍，非常麻烦。

**EST Data 方式**：你告诉档案管理员你要存什么东西，它会自动帮你：
- 保存数据（存档案）
- 查找数据（找档案）
- 更新数据（改档案）
- 删除数据（丢档案）

它支持多种存储方式：内存存储、数据库存储、Redis 缓存、MongoDB 文档存储，想用哪种用哪种！

### 核心特点

- 🎯 **简单易用** - 几行代码就能完成 CRUD 操作
- 🔌 **多数据源支持** - 内存、JDBC、Redis、MongoDB 一网打尽
- 🔒 **类型安全** - 泛型支持，编译时就能发现类型错误
- 🎨 **Repository 模式** - 统一的接口，切换数据源不用改代码
- ⚡ **高性能** - 内置缓存和优化，快速存取数据

### 第一步：引入依赖

在你的 Maven pom.xml 中添加：

```xml
<dependencies>

        <artifactId>est-data-api</artifactId>
        <version>2.0.0</version>
    </dependency>
    <dependency>
        <groupId>ltd.idcu</groupId>
        <artifactId>est-data-memory</artifactId>
        <version>2.0.0</version>
    </dependency>
</dependencies>
```

### 第二步：你的第一个程序

```java
import ltd.idcu.est.data.api.Entity;
import ltd.idcu.est.data.api.Id;
import ltd.idcu.est.data.api.Repository;
import ltd.idcu.est.data.memory.MemoryData;

public class FirstExample {
    public static void main(String[] args) {
        // 1. 创建一个 Repository（档案管理员）
        Repository<User, Long> userRepository = MemoryData.newRepository();
        
        // 2. 创建一个用户
        User user = new User();
        user.setName("张三");
        user.setAge(25);
        
        // 3. 保存用户
        User savedUser = userRepository.save(user);
        System.out.println("保存的用户：" + savedUser);
        
        // 4. 查找用户
        User foundUser = userRepository.findById(1L).orElse(null);
        System.out.println("找到的用户：" + foundUser);
        
        // 5. 查看所有用户
        System.out.println("所有用户：" + userRepository.findAll());
    }
}

@Entity
class User {
    @Id(autoGenerated = true)
    private Long id;
    private String name;
    private int age;
    
    public Long getId() { return id; }
    public void setId(Long id) { this.id = id; }
    public String getName() { return name; }
    public void setName(String name) { this.name = name; }
    public int getAge() { return age; }
    public void setAge(int age) { this.age = age; }
    
    @Override
    public String toString() {
        return "User{id=" + id + ", name='" + name + "', age=" + age + "}";
    }
}
```

运行这个程序，你会看到：
```
保存的用户：User{id=1, name='张三', age=25}
找到的用户：User{id=1, name='张三', age=25}
所有用户：[User{id=1, name='张三', age=25}]
```

恭喜你！你已经成功使用 EST Data 了！

## 基础篇

### 1. 什么是 Repository？

Repository 就是一个"数据仓库"接口，它定义了标准的 CRUD（增删改查）操作。

不管你用的是内存存储、数据库还是 Redis，Repository 的接口都是一样的！

#### Repository 的核心方法

```java
public interface Repository<T, ID> {
    T save(T entity);                    // 保存单个实体
    Iterable<T> saveAll(Iterable<T> entities);  // 批量保存
    
    Optional<T> findById(ID id);        // 根据 ID 查找
    List<T> findAll();                   // 查找所有
    List<T> findAllById(Iterable<ID> ids);  // 批量查找
    
    boolean existsById(ID id);           // 检查是否存在
    long count();                        // 统计数量
    
    void deleteById(ID id);              // 根据 ID 删除
    void delete(T entity);               // 删除单个实体
    void deleteAll();                    // 删除所有
}
```

### 2. 定义实体类

要使用 Repository，首先需要定义一个实体类，就像定义一个"档案模板"。

```java
import ltd.idcu.est.data.api.Entity;
import ltd.idcu.est.data.api.Id;

@Entity  // 标记这是一个实体类
class Product {
    @Id(autoGenerated = true)  // 标记主键，自动生成
    private Long id;
    
    private String name;
    private double price;
    private int stock;
    
    // Getter 和 Setter
    public Long getId() { return id; }
    public void setId(Long id) { this.id = id; }
    public String getName() { return name; }
    public void setName(String name) { this.name = name; }
    public double getPrice() { return price; }
    public void setPrice(double price) { this.price = price; }
    public int getStock() { return stock; }
    public void setStock(int stock) { this.stock = stock; }
    
    @Override
    public String toString() {
        return "Product{id=" + id + ", name='" + name + "', price=" + price + ", stock=" + stock + "}";
    }
}
```

### 3. 基本 CRUD 操作

让我们用 Product 实体来演示基本的增删改查操作：

```java
import ltd.idcu.est.data.api.Repository;
import ltd.idcu.est.data.memory.MemoryData;

import java.util.Arrays;
import java.util.List;

public class BasicCrud {
    public static void main(String[] args) {
        Repository<Product, Long> repo = MemoryData.newRepository();
        
        System.out.println("--- 1. 新增（Create）---");
        Product p1 = new Product();
        p1.setName("iPhone 15");
        p1.setPrice(5999.0);
        p1.setStock(100);
        repo.save(p1);
        
        Product p2 = new Product();
        p2.setName("MacBook Pro");
        p2.setPrice(14999.0);
        p2.setStock(50);
        repo.save(p2);
        
        System.out.println("保存了 2 个产品");
        
        System.out.println("\n--- 2. 查询（Read）---");
        Product found = repo.findById(1L).orElse(null);
        System.out.println("找到 ID=1 的产品：" + found);
        
        List<Product> all = repo.findAll();
        System.out.println("所有产品：" + all);
        
        System.out.println("\n--- 3. 更新（Update）---");
        Product toUpdate = repo.findById(1L).orElse(null);
        if (toUpdate != null) {
            toUpdate.setPrice(5499.0);  // 降价促销！
            toUpdate.setStock(80);
            repo.save(toUpdate);
            System.out.println("更新后的产品：" + repo.findById(1L).orElse(null));
        }
        
        System.out.println("\n--- 4. 删除（Delete）---");
        repo.deleteById(2L);
        System.out.println("删除后剩余产品数：" + repo.count());
        
        repo.deleteAll();
        System.out.println("清空后剩余产品数：" + repo.count());
    }
}
```

## 进阶篇

### 1. 批量操作

Repository 支持批量保存和批量删除，效率更高！

```java
import ltd.idcu.est.data.api.Repository;
import ltd.idcu.est.data.memory.MemoryData;

import java.util.Arrays;
import java.util.List;

public class BatchOperations {
    public static void main(String[] args) {
        Repository<Product, Long> repo = MemoryData.newRepository();
        
        System.out.println("--- 批量保存 ---");
        Product p1 = new Product();
        p1.setName("产品A");
        p1.setPrice(100.0);
        p1.setStock(10);
        
        Product p2 = new Product();
        p2.setName("产品B");
        p2.setPrice(200.0);
        p2.setStock(20);
        
        Product p3 = new Product();
        p3.setName("产品C");
        p3.setPrice(300.0);
        p3.setStock(30);

        System.out.println("批量保存了 " + repo.count() + " 个产品");
        
        System.out.println("\n--- 批量查找 ---");
        List<Product> found = repo.findAllById(Arrays.asList(1L, 3L));
        System.out.println("找到 ID=1 和 3 的产品：" + found);
        
        System.out.println("\n--- 批量删除 ---");
        repo.deleteAllById(Arrays.asList(1L, 2L));
        System.out.println("删除后剩余产品数：" + repo.count());
    }
}
```

### 2. 使用 MemoryRepository 的高级查询

MemoryRepository 还支持基于 Predicate 的灵活查询：

```java
import ltd.idcu.est.data.memory.MemoryData;
import ltd.idcu.est.data.memory.MemoryRepository;

import java.util.List;
import java.util.Optional;

public class AdvancedQuery {
    public static void main(String[] args) {
        MemoryRepository<Product, Long> repo = MemoryData.newRepository();
        
        // 先添加一些测试数据
        Product p1 = new Product();
        p1.setName("iPhone 15");
        p1.setPrice(5999.0);
        p1.setStock(100);
        repo.save(p1);
        
        Product p2 = new Product();
        p2.setName("iPhone 15 Pro");
        p2.setPrice(8999.0);
        p2.setStock(50);
        repo.save(p2);
        
        Product p3 = new Product();
        p3.setName("MacBook Pro");
        p3.setPrice(14999.0);
        p3.setStock(30);
        repo.save(p3);
        
        Product p4 = new Product();
        p4.setName("iPad");
        p4.setPrice(3999.0);
        p4.setStock(0);  // 缺货
        repo.save(p4);
        
        System.out.println("--- 查找价格小于 10000 的产品 ---");
        List<Product> affordable = repo.findByPredicate(p -> p.getPrice() < 10000);
        affordable.forEach(System.out::println);
        
        System.out.println("\n--- 查找有库存的产品 ---");
        List<Product> inStock = repo.findByPredicate(p -> p.getStock() > 0);
        inStock.forEach(System.out::println);
        
        System.out.println("\n--- 查找名称包含 'iPhone' 的产品 ---");
        List<Product> iphones = repo.findByPredicate(p -> p.getName().contains("iPhone"));
        iphones.forEach(System.out::println);
        
        System.out.println("\n--- 查找第一个价格大于 5000 的产品 ---");
        Optional<Product> expensive = repo.findFirstByPredicate(p -> p.getPrice() > 5000);
        expensive.ifPresent(p -> System.out.println("找到：" + p));
        
        System.out.println("\n--- 统计价格大于 5000 的产品数量 ---");
        long count = repo.countByPredicate(p -> p.getPrice() > 5000);
        System.out.println("数量：" + count);
        
        System.out.println("\n--- 删除所有缺货的产品 ---");
        repo.deleteByPredicate(p -> p.getStock() == 0);
        System.out.println("删除后剩余：" + repo.count() + " 个产品");
    }
}
```

### 3. 使用 Builder 自定义 MemoryRepository

你可以使用 Builder 来自定义 ID 生成器和 ID 提取器：

```java
import ltd.idcu.est.data.memory.MemoryData;
import ltd.idcu.est.data.memory.MemoryRepository;

import java.util.UUID;

public class CustomRepository {
    public static void main(String[] args) {
        MemoryRepository<Product, String> repo = MemoryData.<Product, String>repositoryBuilder()
            .idGenerator(() -> UUID.randomUUID().toString())  // 使用 UUID 作为 ID
            .idExtractor(Product::getId)                        // 如何从实体中提取 ID
            .idSetter((product, id) -> {                        // 如何设置 ID 到实体
                product.setId(id);
                return product;
            })
            .build();
        
        Product product = new Product();
        product.setName("自定义 ID 产品");
        product.setPrice(999.0);
        product.setStock(10);
        
        Product saved = repo.save(product);
        System.out.println("保存的产品（UUID ID）：" + saved);
    }
}
```

## 高级篇

### 1. 事务管理

MemoryTransactionManager 提供了简单的事务支持：

```java
import ltd.idcu.est.data.api.Repository;
import ltd.idcu.est.data.memory.MemoryData;
import ltd.idcu.est.data.memory.MemoryTransactionManager;

public class TransactionExample {
    public static void main(String[] args) {
        Repository<Product, Long> repo = MemoryData.newRepository();
        MemoryTransactionManager txManager = MemoryData.newTransactionManager();
        
        System.out.println("--- 事务示例 ---");
        
        try {
            txManager.begin();
            
            Product p1 = new Product();
            p1.setName("产品1");
            p1.setPrice(100.0);
            p1.setStock(10);
            repo.save(p1);
            
            Product p2 = new Product();
            p2.setName("产品2");
            p2.setPrice(200.0);
            p2.setStock(20);
            repo.save(p2);
            
            txManager.commit();
            System.out.println("事务提交成功，保存了 " + repo.count() + " 个产品");
        } catch (Exception e) {
            txManager.rollback();
            System.out.println("事务回滚");
        }
    }
}
```

## 与 est-collection 集成

EST Data 和 est-collection 是绝配！让我们看看它们如何配合使用。

### 场景：电商产品管理系统

```java
import ltd.idcu.est.collection.api.Seq;
import ltd.idcu.est.collection.impl.Seqs;
import ltd.idcu.est.data.api.Repository;
import ltd.idcu.est.data.memory.MemoryData;
import ltd.idcu.est.data.memory.MemoryRepository;

import java.util.List;

public class CollectionIntegration {
    public static void main(String[] args) {
        MemoryRepository<Product, Long> repo = MemoryData.newRepository();
        
        // 先添加一些测试数据
        Product p1 = new Product();
        p1.setName("iPhone 15");
        p1.setPrice(5999.0);
        p1.setStock(100);
        repo.save(p1);
        
        Product p2 = new Product();
        p2.setName("iPhone 15 Pro");
        p2.setPrice(8999.0);
        p2.setStock(50);
        repo.save(p2);
        
        Product p3 = new Product();
        p3.setName("MacBook Pro");
        p3.setPrice(14999.0);
        p3.setStock(30);
        repo.save(p3);
        
        Product p4 = new Product();
        p4.setName("iPad");
        p4.setPrice(3999.0);
        p4.setStock(0);
        repo.save(p4);
        
        Product p5 = new Product();
        p5.setName("AirPods Pro");
        p5.setPrice(1999.0);
        p5.setStock(200);
        repo.save(p5);
        
        System.out.println("=== EST Data + EST Collection 集成示例 ===\n");
        
        // 1. 从 Repository 获取数据，转换为 Seq
        List<Product> productList = repo.findAll();
        Seq<Product> products = Seqs.from(productList);
        
        System.out.println("--- 1. 筛选有库存且价格 < 10000 的产品 ---");
        products
            .where(p -> p.getStock() > 0)
            .where(p -> p.getPrice() < 10000)
            .forEach(System.out::println);
        
        System.out.println("\n--- 2. 按价格从低到高排序 ---");
        products
            .sortBy(Product::getPrice)
            .forEach(System.out::println);
        
        System.out.println("\n--- 3. 计算所有有库存产品的总价值 ---");
        double totalValue = products
            .where(p -> p.getStock() > 0)
            .mapToDouble(p -> p.getPrice() * p.getStock())
            .sum();
        System.out.println("库存总价值：" + totalValue + " 元");
        
        System.out.println("\n--- 4. 找出最贵的 3 个产品 ---");
        products
            .sortBy(Product::getPrice, true)
            .take(3)
            .forEach(System.out::println);
        
        System.out.println("\n--- 5. 统计各价格区间的产品数量 ---");
        System.out.println("价格 < 5000：" + products.where(p -> p.getPrice() < 5000).count());
        System.out.println("5000-10000：" + products.where(p -> p.getPrice() >= 5000 && p.getPrice() < 10000).count());
        System.out.println("价格 >= 10000：" + products.where(p -> p.getPrice() >= 10000).count());
        
        System.out.println("\n--- 6. 组合操作：找出有库存的 iPhone 并涨价 10% ---");
        products
            .where(p -> p.getStock() > 0)
            .where(p -> p.getName().contains("iPhone"))
            .map(p -> {
                p.setPrice(p.getPrice() * 1.1);
                return p;
            })
            .forEach(p -> {
                repo.save(p);
                System.out.println("更新后：" + p);
            });
    }
}
```

## 最佳实践

### 1. 为每个实体创建专用 Repository 接口

```java
// ✅ 推荐：创建专用 Repository
interface ProductRepository extends Repository<Product, Long> {
    List<Product> findByPriceLessThan(double price);
    List<Product> findByStockGreaterThan(int stock);
}

// 使用时
ProductRepository repo = ...;
List<Product> cheapProducts = repo.findByPriceLessThan(1000);
```

### 2. 合理使用缓存

对于频繁查询但很少变动的数据，可以结合 est-cache 使用：

```java
// 先查缓存，缓存没有再查数据库
Product product = cache.get(id);
if (product == null) {
    product = repository.findById(id).orElse(null);
    if (product != null) {
        cache.put(id, product);
    }
}
```

### 3. 批量操作优于单条操作

```java
// ✅ 推荐：批量操作
List<Product> products = ...;
repository.saveAll(products);

// ❌ 不推荐：循环单条操作
for (Product product : products) {
    repository.save(product);
}
```

### 4. 使用 Optional 处理查找结果

```java
// ✅ 推荐：使用 Optional
Optional<Product> optional = repository.findById(id);
optional.ifPresent(product -> {
    System.out.println("找到产品：" + product);
});

// 或提供默认值
Product product = optional.orElse(defaultProduct);
```

## 常见问题

### Q: 内存存储的数据会持久化吗？

A: MemoryRepository 是内存存储，程序重启后数据会丢失。如果需要持久化，请使用 JDBC、MongoDB 或 Redis 实现。

### Q: 如何切换不同的数据源？

A: 只需要改变 Repository 的实现类，业务代码不需要修改。这就是 Repository 模式的优势！

### Q: 支持复杂查询吗？

A: MemoryRepository 支持基于 Predicate 的灵活查询。对于 JDBC 和 MongoDB 实现，还支持更复杂的查询语法。
