# EST 项目文档更新指南 - AI 编码助手

本文档提供给 AI 编码助手使用，指导如何按照统一的风格更新 EST 项目的所有文档。

## 文档更新原则

### 1. 小白友好原则
- 从基础概念讲起，避免假设读者有高级知识
- 使用生活中的例子类比技术概念
- 循序渐进，从简单到复杂
- 提供大量完整的可运行示例

### 2. 递进式结构
每个模块的文档应该包含以下章节：
```
## 快速入门：5分钟上手
## 基础篇
## 进阶篇
## 高级篇
## 最佳实践
## 常见问题
```

### 3. 内容详实原则
- 每个功能都要有代码示例
- 示例要有注释说明
- 展示输入和预期输出
- 提供完整的端到端案例

### 4. 与 est-collection 联动
所有涉及数据处理的文档都应该：
- 展示如何与 est-collection 配合使用
- 提供链式操作的示例
- 展示数据从获取到处理到输出的完整流程

### 示例代码标准

1. **每个示例都应该是完整的**
   - 包含必要的 import
   - 包含 main 方法
   - 可以直接运行

2. **代码应该有注释**
   ```java
   // 这是一个示例注释
   Seq<User> users = Seqs.of(...);
   
   // 筛选活跃用户
   Seq<User> activeUsers = users.where(User::isActive);
   ```

3. **示例应该展示输入和输出**
   ```java
   System.out.println("结果：" + result);
   // 输出：结果：[a, b, c]
   ```

## 集成示例标准

### 与 est-collection 集成的模式

```java
// 1. 从其他模块获取数据
List<Data> dataList = someModule.getData();

// 2. 转换为 Seq
Seq<Data> dataSeq = Seqs.from(dataList);

// 3. 使用 est-collection 处理数据
Seq<Result> result = dataSeq
    .where(...)
    .map(...)
    .sortBy(...);

// 4. 转换回其他格式使用
someModule.process(result.toList());
```

## 注意事项

1. **不要过度设计** - 保持文档简洁，重点在示例
2. **先写示例，再写解释** - 代码是最好的文档
3. **保持一致性** - 所有模块的文档风格要统一
4. **确保示例可运行** - 所有代码示例都应该经过测试
