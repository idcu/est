# EST AI 常见问题

## 一般问题

### Q: EST 框架是什么？
A: EST 是一个专为 AI 时代设计的 Java 全栈开发框架，它将现代化的 Java 开发实践与强大的 AI 能力完美融合，让开发者能够高效地构建智能应用。

### Q: EST 是免费的吗？
A: 是的！EST 框架本身是开源免费的，采用 Apache 2.0 许可证。使用 LLM 服务可能需要支付相应提供商的费用，但大多数提供商都提供免费额度。

### Q: EST 适用于生产环境吗？
A: 当然！EST 框架是为企业级应用设计的，具有完善的安全机制、错误处理和性能优化，完全适用于生产环境。

## 安装和配置

### Q: 如何安装 EST？
A: 最简单的方式是通过 Maven 或 Gradle 添加依赖。详细步骤请参考 [快速入门](./quickstart.md)。

### Q: 支持哪些 JDK 版本？
A: EST 框架需要 JDK 17 或更高版本。

### Q: 如何配置 LLM API Key？
A: 有两种方式：
1. 编程方式：通过 `LlmConfig` 设置
2. Spring Boot 配置方式：在 `application.yml` 中配置

详细说明请参考 [快速入门](./quickstart.md)。

### Q: 支持哪些 LLM 提供商？
A: EST 支持多种主流 LLM 提供商：
- OpenAI (GPT-4, GPT-3.5)
- 智谱 AI (GLM-4, GLM-3)
- 通义千问 (Qwen-Turbo, Qwen-Plus)
- 文心一言 (Ernie-4.0, Ernie-3.5)
- 豆包 (Doubao-Pro)
- Kimi (Moonshot-v1)
- Ollama (本地模型)

## AI 功能

### Q: 代码生成器支持生成哪些代码？
A: 代码生成器支持生成：
- Entity 实体类
- Repository 数据访问层
- Service 业务逻辑层
- Controller 控制层
- pom.xml 项目配置文件

### Q: AI 助手能做什么？
A: AI 助手提供以下功能：
- 自然语言对话
- 获取快速参考
- 获取最佳实践
- 获取教程
- 代码建议
- 代码解释
- 代码优化

### Q: 提示词模板有什么用？
A: 提示词模板可以帮助你：
- 确保 AI 输出质量的一致性
- 减少重复编写提示词的工作
- 提供专业级的提示词
- 支持自定义模板

### Q: 可以在没有网络的情况下使用吗？
A: EST 支持使用 Ollama 等本地模型，可以在没有网络连接的情况下使用。

## 技术问题

### Q: 如何处理 LLM API 调用超时？
A: EST 内置了超时和重试机制。你可以：
1. 配置合理的超时时间
2. 使用重试机制
3. 考虑使用缓存

详细内容请参考 [最佳实践](./best-practices.md)。

### Q: 如何确保 AI 生成代码的安全性？
A: 建议采取以下措施：
1. 人工审查所有生成的代码
2. 使用静态代码分析工具
3. 运行完整的测试套件
4. 在沙箱环境中测试

### Q: 如何缓存 LLM 响应以提高性能？
A: 可以使用 Caffeine 等缓存库来缓存 LLM 响应。详细示例请参考 [最佳实践](./best-practices.md)。

### Q: 如何自定义 AiAssistant 的行为？
A: 你可以实现 `AiAssistant` 接口来创建自定义的 AI 助手。详细说明请参考 [架构设计](./architecture.md)。

## 架构和扩展

### Q: 如何添加新的 LLM 提供商支持？
A: 你可以：
1. 实现 `LlmClient` 接口
2. 将你的实现注册到 `LlmClientFactory`

详细说明请参考 [架构设计](./architecture.md)。

### Q: EST AI 模块与其他 EST 模块的关系是什么？
A: EST AI 模块是 EST 框架的一个核心模块，它可以与其他 EST 模块（如 Web、数据访问等）无缝集成。

## 故障排除

### Q: 遇到 "API Key 无效" 错误怎么办？
A: 请检查：
1. API Key 是否正确
2. API Key 是否过期
3. 是否有足够的额度
4. 网络连接是否正常

### Q: LLM 响应很慢怎么办？
A: 可以尝试：
1. 使用更快的模型
2. 减少提示词长度
3. 使用缓存
4. 检查网络连接

### Q: 生成的代码有错误怎么办？
A: 建议：
1. 仔细审查代码
2. 使用 AI 助手解释和优化代码
3. 提供更详细的需求
4. 采用迭代式改进
