# 贡献指南

感谢您对EST框架的关注和支持！我们欢迎并鼓励社区贡献。本指南将帮助您了解如何为EST框架贡献代码。

## 贡献方式

您可以通过以下方式为EST框架贡献：

1. **报告问题**：在GitHub Issues中报告bug或提出新功能建议
2. **提交代码**：通过Pull Request提交代码修改
3. **改进文档**：完善框架文档，包括API文档、用户指南等
4. **分享经验**：在社区中分享使用EST框架的经验和最佳实践

## 开发环境设置

### 环境要求

- JDK 21+
- Maven 3.6+
- Git

### 克隆仓库

```bash
git clone https://github.com/idcu/est.git
cd est
```

### 构建项目

```bash
mvn clean install
```

## 代码规范

### 编码风格

- 遵循Java标准编码规范
- 使用4个空格进行缩进
- 类名使用大驼峰命名法（PascalCase）
- 方法名和变量名使用小驼峰命名法（camelCase）
- 常量使用全大写，单词间用下划线分隔

### 提交规范

- 提交消息应该清晰、简洁，描述本次提交的内容
- 提交消息格式：`[模块名] 简短描述`
- 例如：`[est-core] 修复依赖注入容器的线程安全问题`

## 提交Pull Request

1. **Fork仓库**：在GitHub上fork EST仓库
2. **创建分支**：在您的fork中创建一个新分支
3. **编写代码**：实现功能或修复bug
4. **运行测试**：确保所有测试通过
5. **提交代码**：提交您的修改
6. **创建Pull Request**：从您的分支向主仓库创建Pull Request

## 代码审查

所有Pull Request都会经过代码审查，确保代码质量和一致性。请耐心等待审查结果，并根据审查意见进行修改。

## 行为准则

参与EST框架的贡献请遵循以下行为准则：

- 尊重他人，友善交流
- 接受建设性批评
- 专注于项目的最佳利益
- 对其他贡献者表示感谢
