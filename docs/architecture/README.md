## 架构概览

### 一句话总结

EST 是一个**分层的、模块化的、AI 友好的、云原生的、渐进式学习的 Java 框架，就像搭积木一样，你可以按需组合模块来构建你的应用。

### 整体架构图
```
┌─────────────────────────────────────────────────────────────────┐
│                        应用层（est-app）                       │
│  直接给用户用的"成品工具"，比如 Web 框架、控制台应用、微服务框架  │
├─────────────────────────────────────────────────────────────────┤
│                        模块层（est-modules）                   │
│  各种"功能工具"：缓存、日志、数据库、安全认证、消息队列、监控...   │
├─────────────────────────────────────────────────────────────────┤
│                        核心层（est-core）                      │
│  "基础设施"：依赖注入容器、配置管理、生命周期管理... 所有模块的基础│
├─────────────────────────────────────────────────────────────────┤
│                        基础层（est-base）                      │
│  "底层零件"：工具类、集合增强、设计模式、测试框架...              │
└─────────────────────────────────────────────────────────────────┘
```

### 为什么这样设计？

**类比理解**：
- 基础层 = 砖块、水泥（建筑材料）
- 核心层 = 框架结构、水电管线（基础设施）
- 模块层 = 空调、电梯、消防系统（各种设备）
- 应用层 = 精装修的房子（成品）

**好处**：
1. **按需使用**：想用什么功能，就引入什么模块
2. **易于扩展**：添加新功能不影响现有代码
3. **易于维护**：分层清晰，问题定位容易
4. **易于学习**：可以从简单到复杂，逐步学习
5. **云原生友好**：零依赖设计完美支持 GraalVM 原生镜像
6. **AI 友好**：清晰的模块结构让 AI 更容易理解和生成代码

## 设计原则

EST 框架的设计遵循以下核心原则：

### 1. 模块化（Modularity）
**原则**：每个模块只做一件事，并且把它做好。
**为什么**？
- 易于理解：每个模块职责清晰
- 易于测试：可以单独测试每个模块
- 易于替换：换掉一个模块不影响其他模块

**类比**：就像工具箱里的工具，锤子只负责敲钉子，螺丝刀只负责拧螺丝。

---

### 2. 接口与实现分离（Interface-Implementation Separation）
**原则**：每个模块都有 api（接口）和 impl（实现）两部分。
**为什么**？
- 接口稳定：定义了"做什么"，不经常变
- 实现灵活：定义了"怎么做"，可以有多种实现
- 易于替换：想换实现时，只需要改依赖，不用改代码

**例子**：
```xml
<!-- 接口 - 稳定不变 -->
<dependency>
    <groupId>ltd.idcu</groupId>
    <artifactId>est-cache-api</artifactId>
</dependency>

<!-- 实现 - 可以替换 -->
<dependency>
    <groupId>ltd.idcu</groupId>
    <artifactId>est-cache-memory</artifactId>
</dependency>
```

---

### 3. 下层不依赖上层（No Upward Dependencies）
**原则**：基础层不依赖核心层，核心层不依赖模块层，模块层不依赖应用层。
**为什么**？
- 下层稳定：下层代码变动少
- 上层灵活：上层可以自由变化
- 易于维护：修改上层不影响下层

**类比**：就像建房子，地基不需要知道上面要盖什么户型。

---

### 4. 约定优于配置（Convention Over Configuration）
**原则**：提供合理的默认值，减少配置的需要。
**为什么**？
- 快速上手：新手不用配置很多东西就能用
- 减少错误：默认值通常是最佳实践
- 保持一致：大家用相同的约定，代码风格一致
**例子**：
```java
// ✅ 推荐：用默认配置，简单快速
Cache<String, String> cache = Caches.newMemoryCache();

// 也可以自定义配置
Cache<String, String> customCache = Caches.<String, String>builder()
    .maxSize(10000)
    .defaultTtl(3600000)
    .build();
```

---

### 5. 渐进式学习（Progressive Learning）
**原则**：从简单到复杂，让学习者可以逐步掌握。
**为什么**？
- 小白友好：不会一上来就被复杂概念吓到
- 有成就感：每一步都能看到成果
- 深入理解：先会用，再理解原理

**学习路径**：
1. 入门级：用默认配置，快速跑通
2. 进阶级：学习配置，优化性能
3. 高级级：理解原理，解决复杂问题
4. 专家级：架构设计，扩展框架

---

### 6. 零依赖（Zero Dependencies）
**原则**：仅使用 Java 标准库，不依赖任何第三方库。
**为什么**？
- 无依赖冲突：不用担心版本不兼容问题
- 部署包小：应用体积小，启动快
- 易于理解：代码可维护性高
- 云原生友好：完美支持 GraalVM 原生镜像编译

---

### 7. AI 友好（AI-Friendly）
**原则**：API 设计简洁直观，模块结构清晰，便于 AI 理解和生成代码。
**为什么**？
- AI 更容易学习和使用
- 生成的代码更简洁可读
- 清晰的模块结构让 AI 能快速理解项目
