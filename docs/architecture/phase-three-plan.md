# EST 框架阶段三实施计划

## 概述

阶段三的目标是完善EST框架的企业级能力，包括前后端联调、认证授权、日志管理、监控管理以及第三方集成。这将使EST框架真正具备生产环境可用的完整功能。

---

## 阶段三目标

### 核心目标
1. **前后端联调**：完善后端API，实现前后端完全打通
2. **认证授权**：实现JWT认证和完整的权限控制体系
3. **日志管理模块**：操作日志、登录日志的记录和查询
4. **监控管理模块**：服务监控、在线用户、缓存监控
5. **第三方集成**：支付、短信、邮件、对象存储集成

### 技术栈
- **后端认证**：JWT + EST Security模块
- **日志**：EST Logging模块
- **监控**：EST Monitor模块
- **第三方SDK**：EST Modules中的相应模块

## 高优先级任务

### 1. 前后端联调完善 ⏳

#### 1.1 后端API完善
**认证API**
```java
POST /admin/api/auth/login          # 登录
POST /admin/api/auth/logout         # 登出
GET  /admin/api/auth/current        # 获取当前用户信息
POST /admin/api/auth/refresh-token  # 刷新Token
```

**用户管理API完善**
```java
PUT /admin/api/users/{id}/password  # 修改密码
PUT /admin/api/users/{id}/roles     # 分配角色
```

**角色管理API完善**
```java
PUT /admin/api/roles/{id}/menus     # 分配菜单
```

#### 1.2 前端认证集成
- 登录流程集成（调用后端API）
- Token管理（存储、刷新、过期处理）
- 路由守卫（未登录跳转登录页）
- 请求拦截器（自动添加Token）
- 响应拦截器（401处理）

#### 1.3 动态菜单和权限控制
- 根据用户权限动态加载路由
- 按钮级权限控制（v-permission指令）
- 菜单根据权限动态显示
