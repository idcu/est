# EST 文档模板库

本目录包含 EST 框架项目使用的各种文档模板。这些模板用于保持项目文档的一致性和专业性。

## 模板列表

### 1. project-README.md.template
**用途**: 项目主 README 文档

**应用场景**: 项目根目录的 README.md

**主要占位符**: 
- `{PROJECT_NAME}` - 项目名称
- `{PROJECT_DESCRIPTION}` - 项目描述
- `{VERSION}` - 版本号
- `{REPO_URL}` - 仓库地址
- 等等...

### 2. core-module-README.md.template
**用途**: 核心模块文档（如 est-core）
**应用场景**: 核心功能模块的详细文档
**特点**:
- 包含"从入门到精通"的完整学习路径
- 分为基础篇、进阶篇、高级篇
- 包含大量代码示例
- 最佳实践和常见问题

### 3. feature-module-README.md.template
**用途**: 功能模块组文档（如 est-modules）
**应用场景**: 包含多个子模块的模块组文档
**特点**:
- 模块概览和子模块介绍
- 快速入门指南
- 最佳实践建议

### 4. sub-module-README.md.template
**用途**: 单个子模块文档
**应用场景**: 具体功能模块的文档
**特点**:
- 简明扼要
- API 说明
- 配置选项表格
- 相关资源链接

### 5. example-README.md.template
**用途**: 示例项目文档

**应用场景**: 示例代码项目的 README

**特点**:
- 功能展示
- 运行说明
- 项目结构说明

### 6. docs-index-README.md.template
**用途**: 文档中心索引

**应用场景**: docs/ 目录的主文档

**特点**:
- 文档导航
- 架构说明
- 相关资源链接

### 7. quickstart.md.template
**用途**: 快速开始指南
**应用场景**: 新手入门教程

**特点**:
- 环境准备
- 项目创建步骤
- 代码编写指导
- 常见问题解答

## 使用指南

### 基本使用步骤

1. **选择合适的模板**: 根据文档类型选择对应的模板文件
2. **复制模板**: 将模板文件复制到目标位置并重命名为 README.md（或其他合适的名称）
3. **替换占位符**: 将模板中的 `{PLACEHOLDER}` 替换为实际内容
4. **填充内容**: 根据实际情况添加或删除章节

5. **代码示例**: 确保所有代码示例都是可运行的

### 占位符说明

所有模板都使用 `{PLACEHOLDER_NAME}` 格式的占位符。常见的占位符包括：

#### 项目信息
- `{PROJECT_NAME}` - 项目名称
- `{PROJECT_SHORT_NAME}` - 项目简称
- `{PROJECT_DESCRIPTION}` - 项目描述
- `{VERSION}` - 版本号
- `{AUTHOR}` - 作者
- `{LICENSE}` - 许可证

#### 路径和链接
- `{REPO_URL}` - 仓库地址
- `{DOCS_PATH}` - 文档路径
- `{EXAMPLES_PATH}` - 示例代码路径
- `{MODULES_PATH}` - 模块路径

#### 代码相关
- `{FIRST_EXAMPLE_CODE}` - 第一个代码示例
- `{EXAMPLE_OUTPUT}` - 示例输出
- `{POM_XML_CONTENT}` - pom.xml 内容

#### 日期和版本
- `{DOC_VERSION}` - 文档版本
- `{LAST_UPDATE}` - 最后更新日期（建议格式：YYYY-MM-DD）

## 最佳实践

### 1. 保持一致性
- 所有文档使用相同的格式和风格
- 使用相同的 emoji 图标
- 保持代码示例风格一致

### 2. 保持简洁
- 避免冗长的描述
- 使用列表简化长段文字
- 提供简洁的代码示例

### 3. 易于理解
- 使用"大白话"解释概念
- 提供生动的比喻
- 从简单到复杂逐步介绍

### 4. 实用性
- 提供可运行的代码示例
- 说明常见问题和解决方案
- 推荐最佳实践

### 5. 可维护性
- 使用模板减少重复工作
- 保持文档与代码同步更新
- 定期检查和更新链接

## 文档风格指南

### 标题层级
```markdown
# 一级标题（项目名称）
## 二级标题（主要章节）
### 三级标题（子章节）
#### 四级标题（小项）
```

### 代码块
- Java 代码使用 ` ```java `
- XML 代码使用 ` ```xml `
- Bash 代码使用 ` ```bash `
- 纯文本使用 ` ``` `

### 列表
- 无序列表使用 `-`
- 有序列表使用数字
- 使用 emoji 增加可读性：✨ 📝 🧪 🎯 🚀 🏗️

### 表格
使用 Markdown 表格格式：
```markdown
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| 值1 | 值2 | 值3 |
```

### 链接
- 内部链接使用相对路径
- 外部链接使用完整 URL
- 链接文本要清晰描述目标内容

## 示例：使用模板创建新文档

假设你要为一个新模块 `est-xxx` 创建文档：

```bash
# 1. 进入模块目录
cd est-modules/est-xxx

# 2. 复制模板
cp ../../.docs-templates/sub-module-README.md.template README.md

# 3. 编辑 README.md，替换占位符
# 打开编辑器，替换所有 {PLACEHOLDER}
```
